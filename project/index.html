IST 363 - Advanced Web Development

Class Project (380pts Total)

The class project is a full semester project. You will begin by dreaming up an idea, and developing a project proposal. Once the proposal has been accepted you will then wireframe out your idea. This will give you an idea of how you project will look and let you work through the flow of your application before you start coding. After the wireframes have been worked through you will code version 1 of you application, this will be your midterm project grade. Version 2 will be an enhanced version of your project and will be submitted for your final project grade.

Project Requirements:

  • Your project idea should be an interactive application, not just a static website
  • Your project should include most of the skills and techniques taught throughout the semester
  • Your project should be innovative, think big! The web is capable of almost anything today. The fun will be discovering what you can accomplish!

What you can do:

  • Write your own code, you may use code from other sources or tutorials as login as credit and a link in given in the form of a code comment
  • You may use a UX framework, such as Bootstrap, Foundation, Tailwind etc
  • You may use JavaScript libraries available on NPM or other sources
  • You may use design influence from other applications, BUT YOU MAY NOT COPY. please list sources of influence in code comments
  • Work together, getting help from other programmers is a big part of application development, help is OK, copying is NOT
  • Use icon libraries, images (with copyright release)

What you can't do:

  • Use a complete template, free or purchased, starter templates are OK.
  • Copy a tutorial that shows you how to create an application, using tutorials to build a feature of your application is ok, copying your entire project from a step by step tutorial is not.

Project Proposal (30pts)

Submit one page project idea to Blackboard, think big! there are lots of tools out there to help you achieve your goals, we will work together throughout the semester to build something amazing!

The professor will provide feedback on your idea and make sure your project has the ability to satisfy all the requirements for the best grade possible.

Your proposal should include a list of any datasources and functionality you may need. Ex: My project will need a database to store... or my project will need a way to login users.

Project Wireframe (50pts)

Your wireframe will be your first pass at how your project will function. The wireframe will include the layout of your application, and the flow and screens of how your application will work. This can be completed using Adobe XD, Figma, or a tool of your choice. Submit a PDF of the design to blackboard.

V1 of your Application (The Midterm, 100pts)

This will be your first pass at your project using the skills and tools taught in class. Most of this version will be a static website, focused more on layout and desing, with very minor user interaction (mostly navigation, hover effects, etc).

Version 1 should satisfy the follow criteria

  • Your application should be in line with your proposal and reflect your wireframe
  • Your application should be a static version of your application, with all completed pages of your wireframe
  • Navigation should work
  • You application should work and look good on all devices using responsive design
  • Hover effects and animations should be utilized to enhance user experience
  • You application should include all forms to satisfy you applications requirements
  • Your application should follow accessability best practices
  • You application should pass HTML validation
  • CSS and JS should be in separate files
  • Hosted on github pages

Project Update (30pts)

A one page document, detailing the plan to get to version 2 of your project. This should include possible libraries, data sources, apis or services that will be used to reach version 2 of your application. This should also incorporate feedback provided by the professor from version 1 of the application.

Version 2 of your Application (The Final Project, 170pts)

This is when your application comes to life! Utilizing the skills taught in the second half of the semester, you will add all the features required to make you application work.

WARNING: you may need to rewrite significant parts of your first version to accomplish your final vision

Version 2 should satisfy the follow criteria

  • Incudes all of the requirements from version 1
  • All features should work, this includes utilizing all required APIs, databases, user authentication, etc
  • JavaScript should be written in a separate file
  • Utilize advanced JavaScript taught in class

Application Ideas

  • Fitness Activity Application - Allows users to login and record fitness activity, also record nutrition facts. (Google Fit integration, authentication)
  • A Recipe App - Allows users to post and review recipes. (Integrate with recipe/nutrition data)
  • A book database - Allow users to post books they have read with reviews, create reading lists. (Integrate with amazon to make some extra cash!)
  • An expenses tracker - Log expenses, tag them (or have categories), List expenses, Have a few graphs (last month / last year). Store the data somewhere
  • Check out this, https://flaviocopes.com/sample-app-ideas for more!
